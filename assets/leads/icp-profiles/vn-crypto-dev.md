# VN Crypto Dev (Audience to Onboard onto Arc)

## TL;DR
Vietnamese developer (Web2→Web3 transition or existing DeFi/NFT dev on BSC/Polygon/Solana). Reads @baobaogemschat Telegram, follows VN crypto Telegram groups, GitHub in English but community trust in Vietnamese. Low Arc awareness; high BSC/Polygon/Solana familiarity. AIG functions as their on-ramp: learn Arc by forking AIG, reading code, tracing tx, getting community support in Vietnamese. Winning this persona = they ship something on Arc citing AIG as their gateway + bring 3–5 other VN devs with them.

## Role in AIG goal
VN crypto devs are the *audience*, not the gatekeeper. By onboarding them onto Arc via AIG:
- **Volume play**: If 10 VN devs ship on Arc, Arc team notices "there's a VN builder cohort now"
- **Community amplification**: Each VN dev → tells 5–10 other VN devs in Telegram → exponential awareness of Arc
- **Ecosystem diversity**: Arc's 100+ partners are mostly Western; Vietnamese builders add geographic / cultural diversity
- **Proof of Arc adoption**: "VN devs are building on Arc" = validation that Arc works for non-English, non-Western markets

Winning VN devs = Bao Bao becomes the de facto *Arc-to-Vietnamese-builders bridge*, which is the VN community moderator role Arc team will eventually recognize + formalize.

## Demographics & Context
- **Age**: 22–35
- **Location**: Vietnam (Ho Chi Minh City, Hanoi, Da Nang, tier-2 cities)
- **Background**: 
  - Web2 dev (3–8 yrs) transitioning to Web3 (attracted by crypto bull market / FOMO)
  - OR existing DeFi/NFT dev (1–3 yrs) on BSC/Polygon/Solana, exploring new chains
  - OR full-time job in tech + crypto building as side project
- **English literacy**: Medium-to-high (can read technical docs; prefers community support in Vietnamese)
- **Tech stack**: JavaScript/TypeScript (Next.js, React), Solidity (basic contracts, not auditors), deployed at least 1 contract on testnet
- **On-chain familiarity**: Intermediate. Has used MetaMask, swapped on Uniswap, understands slippage/gas. Never bridged cross-chain before (CCTP = new concept)
- **Income model**: Some are freelancers; some have day jobs; most testing crypto as hobby or side income
- **Time zone**: Vietnam Standard Time (UTC+7); active 8am–11pm VN time

## Goals & Motivations
- **Primary job-to-be-done**: Ship a working dApp on a new chain to add to resume/portfolio; earn points/rewards/recognition in their community
- **Career win**: "I shipped [project] on Arc and the Arc team featured it" OR "I built a Bot for [Arc use case] and got 100 stars on GitHub"
- **Weekly rhythm**: Learning (2–3 hrs; reads blogs/docs), coding (2–4 hrs; side project), community (1–2 hrs; Telegram groups, Discord)
- **Status incentive**: Local recognition in VN crypto Telegram groups > global Twitter followers (they care what other VN devs think more than Western devs)
- **Peer comparison**: "Did my friend deploy on [Polygon/Solana]? What's the fastest way to ship something similar on Arc?"
- **Earning potential**: Some are chasing incentive programs / bounties (Arc grants, Circle bounties) to fund side project

## Pain Points
- **Arc unknown**: Heard of it via Circle news, but have no idea what CCTP is, what makes Arc different from Polygon/Arbitrum/Optimism, why they should care
- **Language barrier**: Arc docs are English-only; even fluent VN devs lose confidence reading complex technical English under time pressure
- **No VN example**: Can't find any VN dev who has shipped on Arc testnet. Makes it feel risky / unsupported
- **Onboarding friction**: Arc docs explain the *protocol*; don't explain "how to go from Polygon dev to Arc dev in 1 hour"
- **Community trust gap**: Western DevRel advice feels institutional/distant. Want to hear from someone like Bao Bao (fellow VN, building in public, using VN Telegram)
- **Proof-of-work gap**: Most Arc builders they see are big projects (Aave, Morpho, Coinbase). Scared to attempt alone because "what if I fail in front of the whole community?"
- **Stablecoin gap**: Most VN dApps use USDT (centralized, no bridge risk). Don't understand why USDC-native is better (need help connecting to VN context: "USDC = fast settlement for merchants, no Tether drama")

## Behavior & Engagement Signals
- **Discovery channels**:
  - @baobaogemschat Telegram (primary source of trust; if Bao Bao recommends something, they believe it)
  - VN crypto Telegram groups (100–10k members each; word-of-mouth spreads fast)
  - GitHub trending (if linked from Telegram, they'll check)
  - X posts RT'd in Telegram groups (algorithm doesn't matter; peer recommendation does)
  - Dev.to / Medium Vietnamese crypto blogs (less trusted than Telegram, but reads if desperate)

- **Trust-earning evidence** (makes them click GitHub link and try locally):
  - **Vietnamese community signal**: If Bao Bao or trusted VN figure says "I use this," that's credible
  - **Simplicity proof**: "Deploy AIG in 5 minutes on Arc testnet. No advanced Solidity required." (Low barrier = they try)
  - **Working demo link**: Vercel URL that loads instantly, no 404s (builds confidence in quality)
  - **Code they can read**: README in English, but code comments in English too; no cryptic variable names (they trace it)
  - **Helpful GitHub issues**: If they ask a question, Bao Bao responds in Vietnamese within 24 hrs (vs. Western projects that ignore them)
  - **VN dev proof**: If 1–2 other VN devs have forked AIG or asked questions in issues, signals it's "safe for VN devs"

- **Instant disqualifiers**:
  - English-only without translation (even if fluent, makes them feel unwelcome)
  - Hype language without working demo ("Revolutionary stablecoin" + no URL = skip)
  - Needs audited contract / advanced Solidity knowledge ("This is for experts only" = not for them)
  - No GitHub repo (they can't learn from code)
  - Paywall / gatekeeping ("Join our Discord to get testnet faucet URL" = suspicious, not inviting)
  - Assumes Western audience ("Deploy on Ethereum mainnet, then bridge to Arc" = high cost, ignores VN fintech reality)
  - Attacking other chains ("Polygon is garbage") instead of explaining Arc's benefits (they may use Polygon; don't want to feel judged)

- **Decision speed**: Slower than Western builders. Wants to read through issues, see comments from other VN devs, trust the community before starting. 1–2 week evaluation if community signal is good; instant abandon if no VN proof.

## Anti-patterns / Instant Disqualifiers

- **Condescending language**: "Advanced developers only" / "For experienced auditors" (makes them feel excluded)
- **English supremacy tone**: No translation effort; assumes everyone is fluent + comfortable with jargon (triggers distrust)
- **Paywall / gatekeeping**: "Premium access for Discord members" (VN devs expect free + open; paywall = feels exploitative)
- **No working demo**: "Coming soon testnet launch" (they've heard this 100x; show code now)
- **Attacking competition**: "Polygon devs are doing it wrong" (some of them use Polygon; makes them defensive)
- **Scam signals**: "Earn 10x your investment" / "Join presale" (VN devs have been rugged; instant dismiss)
- **No code, all marketing**: Twitter threads with no GitHub link (they don't trust hype)
- **Assumes Western financial infrastructure**: "You can use our payment processor if you live in the US" (excludes VN entirely)

## Cultural Notes

**VN developer psychology:**
- **Trust is personal**: Recommendations from friends/figures like Bao Bao > institutional channels
- **Low English confidence (even when fluent)**: Prefer Vietnamese for emotional support; English for technical specs
- **Community > competition**: Celebrate wins together; help each other (unlike Western builder culture, which is more competitive)
- **Speed + results matter**: "Show working code" > "Show deck." Happy with "good enough" if shipped (vs. Western perfectionism)
- **Remittance + fintech sensitivity**: Merchants care deeply about fees, settlement speed, regulatory trust (Arc's USDC + predictable fees = messaging gold)
- **Mobile-first**: VN devs build for phones, not desktops. UX matters. AIG's Vercel demo must be mobile-responsive (or they assume broken)

**Language & tone:**
- Use "VN" not "Vietnamese" (shorter, insider feel)
- Avoid corporate language ("leverage synergies," "disruptive innovation")
- Use "ship" / "build" language (not "launch," not "deploy") — builders recognize this
- Be direct (not "can you consider..."; say "do X")
- Use emoji sparingly (one per message max) — VN devs find excessive emoji unprofessional / immature
- Signal availability ("ask me," "DM me," "I'll respond in 24 hrs") — accountability builds trust
